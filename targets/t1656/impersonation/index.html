Benign T1656 target — Impersonation

This stands in for a phishing page hosted on an unrelated domain that claims to be a brand's login. The link navigates to a URL combining a protected brand term (paypal) with an auth-context keyword (login/verify) — while the registrable domain is not the brand's. With the attack EDR loaded it raises T1656. Benign + local; nothing runs.

➡ Open the impersonation page (/paypal/login/verify)

Complements T1566.002: that rule inspects the host (lookalike/typosquat); T1656 catches the brand + auth context in the path on a non-brand domain. A /paypal/news link (brand term, no auth keyword) does not fire.
